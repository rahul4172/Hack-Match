1. Executive Summary
HackMatch is a real-time matchmaking platform purpose-built for hackathon participants. It lets builders swipe on potential teammates, filter by tech stack and squad role, and connect through an integrated live chat — all within minutes of signing up. Think Tinder, but instead of dates, you leave with a co-founder for a 36-hour build sprint.
Problem: 68% of hackathon participants who register solo either drop out or significantly underperform due to incomplete or incompatible team composition (Source: Devfolio 2024 Survey). There is no dedicated, fast, skill-aware tool to solve this.
1.1 The Opportunity
Over 5,000 hackathons run globally each year, with 2M+ total participants
Solo registration rates hover between 30–45% on major platforms like Devfolio, Unstop, and Devpost
Existing solutions (Discord servers, WhatsApp groups, Devpost forums) are unstructured and slow
No dedicated matching product exists at this intersection of developer community + real-time UX
1.2 Vision Statement
"Make finding your hackathon squad as fast and intuitive as swiping right — so builders spend time building, not searching."

3. Functional Requirements
3.1 Authentication & Onboarding
GitHub OAuth as primary login (trust signal for developer audience)
Google OAuth as fallback
Onboarding wizard (3 screens): Role selection → Skill tagging → Availability + hackathon preferences
Profile auto-population from GitHub: avatar, username, top languages, public repos
Profile completeness meter — incomplete profiles are shown less
3.2 Discovery & Swipe Engine
Card-stack swipe UI (mobile-first, drag gesture + button fallback)
Swipe right / tap Connect = express interest
Swipe left / tap Pass = hide from deck for 30 days
Mutual match = both users swiped right → unlock chat thread
Ranking: Deck generation logic (priority order):
Users attending the same upcoming hackathon
Complementary skill overlap (user needs what you have)
Geographic proximity or explicit remote-OK flag
Activity recency (active in last 7 days surfaced first)

3.4 Real-Time Chat
Chat unlocks only on mutual match
Message types: text, code snippet (syntax highlighted), link preview, file attachment (max 10 MB)
Read receipts and typing indicators
Message reactions (6 emoji quick-react)
Pinned messages per conversation (up to 3)
Report & block from chat header
Chat archived automatically 30 days after a hackathon's end date (preserves history, stops new messages)
3.5 Profile System
Display name, avatar, one-line tagline, extended bio (280 char limit)
Role badge (primary + secondary role)
Skill tags (max 15, searchable taxonomy)
Hackathon history — past events with win badges
GitHub project showcases (pull 3 pinned repos automatically)
Availability status (manual toggle: Available / Busy / In a Hackathon)
Win counter (self-reported, verified via project link)
3.6 Notifications
In-app: new match, new message, profile view (premium)
Email: daily digest of new matches (opt-in), hackathon deadline reminders
Push (V2): via Firebase Cloud Messaging on mobile

5.2 System Architecture Overview
Client (Next.js on Vercel) → API Layer (Express on Railway) → PostgreSQL + Redis (Supabase + Upstash). Socket.io server runs alongside Express for real-time events. A CDN (Vercel Edge) handles static assets and SSR caching globally.
